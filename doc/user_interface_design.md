# UI設計書 (User Interface Design)

## 1. 概要

本ドキュメントは、Knowledge Graph LLMs アプリケーションのユーザーインターフェース設計を定義する。

### 1.1 アプリケーション基本情報
- **アプリケーション名**: テキストから知識グラフを生成
- **フレームワーク**: Streamlit
- **レイアウト**: Wide（全幅表示）
- **ページ設定**: `app.py:113-118`

### 2.2 サイドバーコンポーネント

#### 2.2.1 LLMモデル選択 (`app.py:122-129`)
- **コンポーネント**: `st.sidebar.selectbox`
- **ラベル**: "LLMモデル:"
- **選択肢**:
  - gpt-4.1-mini (デフォルト)
  - gpt-4.1-nano
  - gpt-4.1
  - gpt-4o-mini
  - gpt-4o
- **実装**: `selected_model = st.sidebar.selectbox("LLMモデル:", model_options, index=0)`

#### 2.2.2 入力方法選択 (`app.py:131-143`)
- **コンポーネント**: `st.sidebar.radio`
- **ラベル**: "入力方法を選択してください:"
- **選択肢**:
  - テキストをアップロード (デフォルト)
  - テキストを入力
- **状態管理**: `st.session_state.input_method`
- **動作**: 選択に応じて不要なセッション状態をクリア

#### 2.2.3 ファイルアップロード (`app.py:160-177`)
- **コンポーネント**: `st.sidebar.file_uploader`
- **表示条件**: 入力方法が「テキストをアップロード」の場合
- **ラベル**: "ファイルをアップロード"
- **対応形式**: `.txt`
- **キー**: `text_upload`
- **生成ボタン**: "知識グラフを生成" (`generate_button_upload`)

#### 2.2.4 テキスト直接入力 (`app.py:178-194`)
- **コンポーネント**: `st.sidebar.text_area`
- **表示条件**: 入力方法が「テキストを入力」の場合
- **ラベル**: "テキストを入力してください"
- **高さ**: 200px
- **キー**: `text_input`
- **生成ボタン**: "知識グラフを生成" (`generate_button_text`)

#### 2.2.5 保存済みグラフ選択 (`app.py:145-156`)
- **コンポーネント**: `st.sidebar.selectbox`
- **ラベル**: "保存済みグラフを表示"
- **データソース**: `out/` ディレクトリの `.html` ファイル
- **ファイル命名規則**: `{timestamp}_{summary}_{model}.html`
  - timestamp: `YYYYMMDD_HHMMSS` 形式
  - summary: テキスト要約（25文字以内）
  - model: 使用したLLMモデル名
- **表示形式**: `{summary}_{model}`
- **ソート順**: タイムスタンプ降順（最新が上）

### 2.3 メインコンテンツエリア

#### 2.3.1 ノード・リレーションシップ一覧 (`app.py:65-111`)
- **コンポーネント**: カスタムHTML (`components.html`)
- **高さ**: 320px
- **実装**: `build_data_html(nodes, relationships)`
- **機能**:
  - ノードリスト: `{id} ({type})` 形式
  - リレーションシップリスト: `{source} -[{type}]-> {target}` 形式
  - Copyボタン: クリップボードにコピー（複数ブラウザ対応）
- **スタイル**:
  - 高さ: 300px
  - スクロール: auto
  - ボーダー: 1px solid #ccc
  - パディング: 10px
  - Copyボタン: 右上に絶対配置

#### 2.3.2 グラフ可視化エリア (`app.py:175-177`, `192-194`, `204-206`)
- **コンポーネント**: カスタムHTML (`components.html`)
- **高さ**: 1000px
- **データソース**:
  - 新規生成時: `generate_knowledge_graph()` の戻り値
  - 履歴表示時: 保存済みHTMLファイル
- **実装**: PyVisで生成されたHTMLをそのまま埋め込み

### 2.4 インタラクション

#### 2.4.1 グラフ生成フロー
```
ユーザー操作                     システム動作
─────────────────────────────────────────────────
1. LLMモデル選択             → session_state更新
2. 入力方法選択              → UIコンポーネント切り替え
3. テキスト入力/ファイル選択  → 入力データ保持
4. "知識グラフを生成"クリック  → スピナー表示
                              → generate_knowledge_graph()呼び出し
                              → ノード・リレーション抽出
                              → グラフ可視化
                              → HTMLファイル保存
                              → JSON保存
5. 生成完了                  → 成功メッセージ表示
                              → ノード一覧表示
                              → グラフ表示
```

#### 2.4.2 履歴表示フロー
```
ユーザー操作                     システム動作
─────────────────────────────────────────────────
1. 保存済みグラフを選択       → ファイルパス取得
                              → JSONファイル読み込み
                              → ノード・リレーション表示
                              → HTMLファイル読み込み
                              → グラフ表示
```

#### 2.4.3 コピー機能 (`app.py:92-108`)
- **トリガー**: Copyボタンクリック
- **対象**: data-box内のテキスト全体
- **実装**: 3段階のフォールバック
  1. Clipboard API (`navigator.clipboard.writeText`)
  2. IE互換 (`window.clipboardData`)
  3. レガシー方式 (`document.execCommand('copy')`)
- **UUID使用理由**: 複数グラフ表示時のID競合回避 (`app.py:72`)

## 3. 状態管理

### 3.1 Session State
| キー | 型 | 用途 | 初期値 |
|------|---|------|--------|
| `input_method` | str | 入力方法の記憶 | "テキストをアップロード" |
| `text_upload` | UploadedFile | アップロードファイル | None |
| `text_input` | str | 直接入力テキスト | None |

### 3.2 状態遷移
- 入力方法切り替え時: 未使用のセッション状態をクリア (`app.py:140-143`)
- グラフ生成完了時: `generated` フラグを `True` に設定 (`app.py:177`, `194`)

## 5. データ変換ユーティリティ

### 5.1 ノード変換 (`app.py:11-31`)
- **関数**: `_to_node_dict(node)`
- **入力**: Node オブジェクトまたは dict
- **出力**: `{"id": str, "type": str}`
- **対応形式**:
  - dict型: `get()` メソッド使用
  - オブジェクト型: `getattr()` 使用

### 5.2 リレーション変換 (`app.py:34-62`)
- **関数**: `_to_rel_dict(rel)`
- **入力**: Relationship オブジェクトまたは dict
- **出力**: `{"source": str, "target": str, "type": str}`
- **対応形式**:
  - dict型: `get()` メソッド使用
  - オブジェクト型: `getattr()` 使用（source.id, target.id）
